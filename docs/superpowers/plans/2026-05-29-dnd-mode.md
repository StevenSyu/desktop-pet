# 勿擾模式（DND）Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 右鍵選單一鍵切換「勿擾模式」，開啟時訊息照進歷史/徽章/通知中心但不彈卡片不演動畫。

**Architecture:** Main 端 gate——ingest `onEvent` 在 send `pet-event` 前用 dnd 旗標擋下。dnd 狀態存 `prefs.json`；window.ts 的 `set-dnd` handler 改 prefs 並經 `bus` 廣播給 index.ts 的本地 `dndEnabled`。通知中心 header 顯示「勿擾中」，pet renderer 收 `dnd-on` 清當前 replay 卡片。

**Tech Stack:** TypeScript, Vitest, Electron ipcMain/contextBridge, EventEmitter bus。

**分支建議：** `feat/dnd-mode`

`loadPrefs` 兩個 return（檔案不存在分支 + try 分支 + catch 分支）都要帶 dnd。整個函式改成：

```ts
export function loadPrefs(userDataDir: string): Prefs {
  const path = join(userDataDir, FILENAME)
  if (!existsSync(path)) {
    return { autoWalk: DEFAULTS.autoWalk, walk: { ...DEFAULTS.walk }, skin: DEFAULTS.skin, dnd: DEFAULTS.dnd }
  }
  try {
    const parsed = JSON.parse(readFileSync(path, 'utf8')) as Record<string, unknown>
    const walkRaw = (parsed.walk ?? {}) as Record<string, unknown>
    return {
      autoWalk: typeof parsed.autoWalk === 'boolean' ? parsed.autoWalk : DEFAULTS.autoWalk,
      walk: sanitizeWalkBounds(walkRaw as Partial<WalkBounds>),
      skin: isValidSkinId(parsed.skin) ? (parsed.skin as string) : DEFAULTS.skin,
      dnd: typeof parsed.dnd === 'boolean' ? parsed.dnd : DEFAULTS.dnd,
    }
  } catch {
    return { autoWalk: DEFAULTS.autoWalk, walk: { ...DEFAULTS.walk }, skin: DEFAULTS.skin, dnd: DEFAULTS.dnd }
  }
}
```

      prefs = { ...prefs, dnd: enabled }
      savePrefs(app.getPath('userData'), prefs)

## 風險與回退

- **dnd 值雙來源不一致**：window.ts 的 `prefs.dnd`（選單顯示用）與 index.ts 的 `dndEnabled`（gate 用）是兩份。靠 `bus.emit('dnd-changed')` 同步——只要 applyDnd 一定 emit 就一致。風險點：若日後有別處改 prefs.dnd 沒 emit，gate 會 stale。本 spec 只有 applyDnd 一個寫入點，安全。
- **getPrefs 型別**：`api.d.ts` 的 `getPrefs` return 型別目前是 `{ autoWalk, walk }`，實際 main 回完整 prefs（含 skin、dnd）。本計畫不動 getPrefs（DND 用獨立 getDnd），型別不一致是既有狀況、不在本 spec 範圍。
- **DND 中 idle 自走仍動**：刻意保留（ambient 行為不算打擾）。若使用者覺得吵，未來可在 DND 時一併停走動——非 v1。
